Cave Explorer

by: Joel Castro

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

Cave Explorer was inspired by flash 2D platformers that I played growing up. I figured it was a project idea that was complicated enough to be challenging, but not so much that it would be super difficult to manage.

2. What I Learned

I learned that people are really good at finding ways to break your game. Their feedback was very helpful in discovering/patching bugs.

3. Architecture of My Game

I organized my code into "screens". There's a "home screen" with buttons to maneuver into any other screen or exit the game, a "setting screen" that allows the user to pick and choose what game sounds they hear, a "game screen" where the actual game takes place and the user plays through a series of levels, and an "end screen" that will pop up upon completing the game which allows users to return to the home screen.

4. Technical Implementation

I am proudest of my implementation of animation in my game. I created a class that takes in a series of images and iterates through them on a counter (according to what the frame rate is set to). This made it really easy to, for example, play a character's running animation only when the user is moving with the arrow keys and not when standing still.

5. Most Challenging Part

The part of the game that was most difficult to implement was the collision logic. I achieved collision physics that I was relatively happy with by, at each frame, having every entity check where it is in relation to the player. If according to a series of if-statements, an entity is overlapping with the player, then the player's x & y velocities are set to 0 (player is stopped) and their position is set to the edge of the entity they are in contact with. Sounds simple enough, but the sheer number of if-statements made this task one of my longest.

6. Future Fixes and Updates

If I were to continue working on this game I'd likely continue to tweak the collision physics because sometimes, when you land on the edge of a block, the player will get pushed off. Apart from that, I would add more content. In the interest of time, I only designed two levels; however, I made level design functions are very intuitive so any future level design would be relatively straightforward.
